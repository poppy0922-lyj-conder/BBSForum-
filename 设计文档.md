## 1. 系统架构

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    用户层 (Browser)                       │
│    Tailwind CSS + Font Awesome + JSP + JavaScript       │
└────────────────────────┬────────────────────────────────┘
                         │ HTTP Request / Response
┌────────────────────────▼────────────────────────────────┐
│                   Web 容器层                              │
│          嵌入式 Tomcat 10.1.52 + Jakarta Servlet 5.0     │
│                                                          │
│    ┌─────────────┐  ┌─────────────┐  ┌───────────────┐  │
│    │  Controller  │  │    Filter   │  │     JSP       │  │
│    │  (Servlet)   │  │  (Auth/Stat │  │  (View层)     │  │
│    │   doGet/Post │  │   /Encoding)│  │  JSTL渲染     │  │
│    └──────┬───────┘  └─────────────┘  └───────────────┘  │
│           │                                                 │
│    ┌──────▼───────┐                                        │
│    │   Service    │                                        │
│    │  (工具类)    │                                        │
│    └──────┬───────┘                                        │
└───────────┼────────────────────────────────────────────────┘
            │ JDBC
┌───────────▼────────────────────────────────────────────────┐
│                    数据库层                                  │
│          MySQL 8.0 + InnoDB (支持事务/外键/行锁)               │
│                                                          │
│     users │ posts │ replies │ categories │ demands        │
│     demand_replies │ score_logs │ daily_checkins         │
│     user_follows │ post_likes │ post_favorites           │
└──────────────────────────────────────────────────────────┘
```

### 1.2 架构模式

**MVC 模式**：

| 层级 | 组件 | 职责 |
|------|------|------|
| **Model** | DBUtil、PostMapper | 数据访问、查询映射 |
| **View** | JSP + JSTL + Tailwind | 界面渲染、数据展示 |
| **Controller** | Servlet（doGet/doPost） | 请求路由、业务逻辑、数据传递 |

### 1.3 关键技术选型

| 技术 | 选择理由 |
|------|----------|
| **Jakarta Servlet 5.0** | Java EE 标准，无需第三方 MVC 框架依赖 |
| **嵌入式 Tomcat** | 部署简单，无需单独安装 Tomcat，Main.java 一键启动 |
| **JSP + JSTL** | 服务端渲染，SEO 友好，与 Servlet 天然集成 |
| **MySQL + JDBC** | 关系型数据库，事务支持良好，行锁支持并发控制 |
| **Tailwind CSS CDN** | 零配置，开发效率高，**统一美观的 UI** |
| **BCrypt** | 密码加密存储，防止明文泄露 |

## 3. 数据流设计

### 3.1 帖子发布流程

```
用户填写表单 → POST /post/create
    → PostServlet.handleCreatePost()
        → 参数校验（标题/内容非空、板块必选）
        → 封面图处理（上传文件 → 保存到 /uploads/）
        → INSERT INTO posts ...
        → 积分操作：UPDATE users SET score = score + 10 WHERE id = ?
        → INSERT INTO score_logs (user_id, score=+10, reason='发帖+10')
        → conn.commit()
    → 重定向到 /post/detail?id=newId
```

### 3.2 需求发布与采纳流程

```
发布：
用户填写表单 → POST /demand/create
    → DemandServlet.createDemand()
        → 参数校验（积分>0、积分≤10000）
        → 积分检查（SELECT score FROM users WHERE id=?）
        → conn.setAutoCommit(false)
        → INSERT INTO demands ...
        → UPDATE users SET score = score - ? WHERE id = ? AND score >= ?
        → INSERT INTO score_logs (user_id, score=-X, reason='发布悬赏需求...')
        → conn.commit()
    → 重定向到 /demand

采纳：
需求发布者点击采纳 → POST /demand/accept (AJAX fetch)
    → DemandServlet.acceptReply()
        → SELECT ... FROM demands WHERE id=? FOR UPDATE
        → 权限校验（发布者本人、状态=open、非自采纳）
        → conn.setAutoCommit(false)
        → UPDATE users SET score = score + ? WHERE id = ?
        → INSERT INTO score_logs (user_id, score=+X, reason='回复被采纳...')
        → UPDATE demands SET status='closed', best_reply_id=? WHERE id=?
        → conn.commit()
    → 返回成功 → location.reload()
```

### 3.3 点赞积分流程

```
用户点击 ♥ 按钮 → POST /interact/like (AJAX fetch)
    → InteractionServlet.doLike()
        → 查询帖子作者（SELECT user_id FROM posts WHERE id=?）
        → 切换点赞状态（INSERT 或 DELETE post_likes）
        → 更新 posts.like_count
        → 如果是点赞（非取消）：给作者 +3 分
            → conn.setAutoCommit(false)
            → UPDATE users SET score = score + 3 WHERE id = authorId
            → INSERT INTO score_logs (user_id, score=+3, reason='帖子被点赞')
            → conn.commit()
    → 返回 JSON {ok, action, count}
```

## 4. 安全设计

### 4.1 密码安全

- 注册和修改密码使用 **BCrypt** 加密存储（`org.mindrot.jbcrypt`）
- 登录验证优先 BCrypt 校验，兼容旧明文（首次登录自动升级）
- 密文格式：`$2a$10$...`（60字符）

### 4.2 Session 安全

- 登录成功后调用 `request.changeSessionId()` 防止 Session Fixation
- Session 中存储用户关键信息（id、username、role）
- 退出登录清除完整 Session

### 4.3 SQL 注入防护

- 所有数据库查询使用 **PreparedStatement** 参数化查询
- 分页参数使用 `LIMIT ? OFFSET ?` + int 类型校验
- 搜索关键词使用参数化 LIKE 查询

### 4.4 权限控制

| 资源 | 访问条件 |
|------|----------|
| `/user/profile*` | 需登录（AuthFilter） |
| `/admin/*` | 需 admin 角色（AuthFilter） |
| 编辑帖子 | 帖子作者 或 管理员 |
| 删除帖子 | 帖子作者 或 管理员 |
| 采纳回复 | 仅需求发布者 |
| 编辑需求 | 仅需求发布者（open状态） |

### 4.5 配置安全

- 数据库密码、API 密钥存储在 `config.properties`
- `config.properties` 已加入 `.gitignore`
- 提供 `config.properties.template` 模板
- 上传文件类型白名单：仅允许 jpg/png/gif/webp

### 4.6 请求安全

- 所有写操作使用 POST 方法
- EncodingFilter 统一 UTF-8 编码
- 图片上传限制 5MB，UUID 重命名

## 5. 并发控制

### 5.1 积分并发

采用数据库行锁 `SELECT ... FOR UPDATE` 防止积分并发异常：

```sql
-- 采纳回复时锁定需求行
SELECT user_id, score, status FROM demands WHERE id = ? FOR UPDATE;

-- 更新用户积分时使用原子操作
UPDATE users SET score = score + ? WHERE id = ?;

-- 扣分时校验积分充足
UPDATE users SET score = score - ? WHERE id = ? AND score >= ?;
```

### 5.2 事务管理

所有积分变更操作使用数据库事务：

```java
conn.setAutoCommit(false);
try {
    // 多个 SQL 操作
    conn.commit();
} catch (Exception e) {
    conn.rollback();
} finally {
    conn.setAutoCommit(true);
}
```

## 6. 缓存策略

### 6.1 板块缓存

- 板块列表存储在 `ServletContext`（应用全局）
- 管理员修改板块后通过 `?refresh=1` 触发刷新
- 应用重启自动重新加载

### 6.2 实时数据缓存

- `StatsFilter` 过滤器缓存论坛统计数据
- 缓存有效期 15 秒
- 过期后自动重新查询数据库

### 6.3 封面图缓存

- CoverServlet 返回 `Cache-Control: max-age=86400`
- 同一 postId 始终生成相同 SVG（确定性算法）

## 8. 封面图生成设计

### 8.1 算法

```
输入：postId, title
1. colors = postId % 10 → 10组渐变配色之一
2. seed = postId ^ 0xABCDEF → 确定性随机种子
3. circles = Random(seed) → 生成 3 个装饰圆形
4. char = title.charAt(0) || '?' → 标题首字符
5. 组装 SVG：渐变矩形背景 + 装饰圆形 + 居中大字
输出：Content-Type: image/svg+xml
```

### 8.2 优先级

```
posts.image_url 非空 → 显示原图
posts.image_url 为空 → 调用 /cover/{id}?title={title} 生成 SVG
原图加载失败（onerror）→ 隐藏图片区域
```

## 9. 错误处理

### 9.1 全局错误页面

- Servlet 500 错误 → 显示"服务器内部错误"友好页面
- AuthFilter 403 错误 → 页面显示禁止访问提示
- 404 资源不存在 → 显示"页面不存在"提示

### 9.2 业务错误处理

| 场景 | 错误处理 |
|------|----------|
| 用户名重复 | 返回注册页显示"用户名已存在" |
| 积分不足 | 发布需求页显示"积分不足"错误 |
| 非作者编辑 | 重定向到详情页，无操作权限 |
| 空搜索关键词 | 自动跳转首页 |
| 采纳已结束需求 | 拒绝操作，显示错误提示 |
| AI 总结失败 | 显示"生成失败，请稍后重试" |

### 9.3 数据校验

- 前端 HTML5 校验（required、maxlength、pattern）
- 后端 Servlet 参数校验（空值、格式、长度）
- 数值字段 NumberFormatException 捕获处理
- 所有用户输入经过 HTML 转义（`escapeHtml()`）

## 12. 前端交互设计

### 12.1 AJAX 交互

以下交互使用 Fetch API 异步请求，无页面刷新：

| 交互 | 请求方式 | 反馈机制 |
|------|----------|----------|
| 关注/取消关注 | POST AJAX | 按钮文本切换（+关注 ↔ 已关注） |
| 点赞/取消点赞 | POST AJAX | 图标切换（空心♥ ↔ 红色♥）+ 计数更新 |
| 收藏/取消收藏 | POST AJAX | 图标切换（空心★ ↔ 实心★）+ 计数更新 |
| AI 总结生成 | POST AJAX | 按钮变为"生成中..."，完成后显示总结框 |
| 采纳回复 | POST AJAX | 弹窗确认 → 加载中 → 页面刷新 |

### 12.2 页面跳转

以下交互使用服务端渲染 + URL 参数反馈：

| 交互 | 跳转方式 | 反馈方式 |
|------|----------|----------|
| 发布帖子 | POST → redirect | 直接跳转到新帖详情页 |
| 登录/注册 | POST → redirect | URL success/error 参数+提示浮层 |
| 编辑资料 | POST → redirect | `?updated=1` 参数+提示浮层 |
| 发布需求 | POST → redirect | 跳转到需求列表 |

### 12.3 通知提示

- 操作成功：绿色浮层（`?success=1`），显示成功图标+消息
- 操作失败：红色浮层（`?error=1`），显示错误图标+消息
- 浮层可点击关闭（×按钮）
- 弹窗系统：统一模态框（蓝色信息/橙色确认/红色错误）

## 13. 性能考虑

### 13.1 数据库层面

- 帖子列表使用 `LIMIT/OFFSET` 分页查询
- `score_logs` 和 `daily_checkins` 表建立唯一索引（uk_user_date）
- 互动表（post_likes, post_favorites）建立联合索引
- `posts` 表的 `title` 和 `content` 字段可考虑添加全文索引

### 13.2 应用层面

- 板块列表缓存到 `ServletContext` 减少重复查询
- 统计数据缓存 15 秒避免频繁查询
- 封面图通过 HTTP 缓存减少重复生成
- 图片上传限制大小和格式，防止资源滥用

### 13.3 前端层面

- Tailwind CSS 通过 CDN 按需加载
- 分页加载减少单页数据量
- AJAX 异步加载减少不必要页面刷新
- 列表使用 `line-clamp` 限制摘要行数

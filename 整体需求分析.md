> **文档版本**：v1.0
> **更新日期**：2026-06-06
> **项目名称**：BBS技术社区论坛系统

---

## 1. 项目概述

### 1.1 项目背景

BBS论坛系统是一个基于 Java Servlet + JSP 技术栈构建的轻量级Web论坛系统。项目面向课程设计场景，团队共5人（1组长+4组员），每人负责从数据库建表到后端接口、前端页面的完整全栈开发。

### 1.2 项目目标

构建一个功能完整、界面美观、安全可靠的BBS技术交流社区，支持帖子发布与讨论、需求悬赏、积分激励、用户互动等核心功能，并引入AI智能总结等创新特性。

### 1.3 技术栈

| 类别 | 实际方案 |
|------|----------|
| 后端框架 | Jakarta Servlet 5.0 + 嵌入式Tomcat 10.1.52 |
| 视图模板 | JSP + JSTL 1.2 |
| 数据库 | MySQL 8.0 + InnoDB（事务/行锁/外键） |
| 前端样式 | Tailwind CSS 3.x CDN |
| 图标库 | Font Awesome 4.7 CDN |
| 密码加密 | jBCrypt 0.4 |
| 构建工具 | Maven |

## 2. 用户角色定义

| 角色 | 权限范围 |
|------|----------|
| **游客**（未登录） | 浏览帖子列表、查看帖子详情、搜索、浏览热度榜 |
| **普通用户**（user） | 游客权限 + 发帖/编辑/删除自己的帖子、回复、发布需求/编辑需求、每日签到、关注/点赞/收藏、查看个人中心、查看积分记录 |

## 3. 功能性需求

### F1: 用户注册与登录

**描述**：用户可以通过注册页面创建账号，使用用户名和密码登录系统。

- 注册需填写用户名、密码、确认密码，可选填联系方式、工作性质、工作地点
- 用户名全局唯一，密码使用 **BCrypt** 加密存储
- 登录成功后 Session 保存用户信息供全站鉴权
- 支持旧明文密码自动升级为 BCrypt 密文（兼容初始测试账号）
- 未登录用户访问个人中心或管理后台时自动跳转登录页

### F2: 帖子发布与管理

**描述**：用户可以发布技术帖子，作者和管理员可以管理帖子。

- 支持 Markdown 格式编辑（加粗、链接、插图工具栏）
- 支持 **封面图片上传**（jpg/png/gif/webp，最大5MB，UUID重命名）
- 支持关键词标签（逗号分隔）
- 发帖时选择所属板块
- 作者和管理员可编辑帖子（标题、内容、板块、关键词、封面）
- 作者和管理员可删除帖子（级联删除回复）

### F3: 帖子展示与列表

**描述**：首页和板块页展示帖子列表。

- 排序规则：置顶优先 → 加精次之 → 按发布时间倒序
- 每页 20 条，支持分页导航
- 帖子卡片显示：封面图、标题、标签（置顶/精华）、摘要、关键词、作者、浏览量、发布时间
- 有 AI 总结的帖子对登录用户显示 🤖 标识
- 无封面图的帖子自动生成 **SVG 彩色封面**

### F4: 帖子详情与回复

**描述**：点击帖子进入详情页查看完整内容并参与讨论。

- 浏览量自动 +1
- 帖子内容 Markdown 渲染为 HTML（XSS 转义）
- 回复列表按时间正序排列，显示楼层号
- 已登录用户可发表回复
- 右侧栏显示相关推荐（最多5篇）

### F5: 用户互动系统

**描述**：用户之间可以通过关注、点赞、收藏进行互动。

- **关注作者**：帖子详情页作者名旁显示关注按钮，AJAX 异步切换
- **点赞帖子**：♥ 按钮，计数实时更新，给帖子作者 +3 积分
- **收藏帖子**：书签按钮，计数实时更新
- 所有互动操作无页面刷新
- 个人中心可查看关注列表并取消关注

### F6: 管理员内容管控

**描述**：管理员可以对帖子进行置顶和加精操作。

- **置顶**：三级状态循环切换（无→板块置顶→全局置顶→无）
- **加精**：开关切换（无↔已加精）
- 所有操作有 **二级确认提示**，避免误操作
- 置顶和加精的帖子在列表中优先级更高

### F7: 板块分类管理

**描述**：管理员可以管理论坛板块分类。

- 板块包含名称、描述、排序权重
- 支持添加、编辑、删除板块
- 用户可在侧边栏浏览板块列表，点击筛选帖子
- 板块列表缓存到全局 ServletContext

### F8: 需求悬赏

**描述**：用户可以发布悬赏需求，其他用户通过回复提供解决方案。

- 发布需求时设置悬赏积分，从发布者账户扣除
- 需求状态：进行中（open）、已结束（closed）
- 发布者可以 **采纳** 一条回复为最佳答案
- 采纳后积分转移给被采纳者，需求自动结束
- 已结束的需求列表页灰色显示，回复区锁定
- 发布者可编辑自己的需求（仅进行中状态）
- 回复存入独立的需求回复表（与帖子回复分离）

### F9: 积分系统

**描述**：系统提供完整的积分激励机制。

| 操作 | 积分变动 | 说明 |
|------|:--------:|------|
| 发布帖子 | **+10** | 发帖成功后自动增加 |
| 回复帖子 | **+2** | 回复成功后给回复者增加 |
| 每日签到 | **+5 ~ +15** | 每天一次，连续递增封顶+15 |
| 帖子被点赞 | **+3** | 给帖子作者增加 |
| 每日首次登录 | **+2** | 当天首次登录发放 |
| 回复被采纳 | **+悬赏分** | 发布者采纳后转给回复者 |
| 发布需求 | **-悬赏分** | 发布时扣除，需积分充足 |

- 所有积分变动通过 `score_logs` 表记录流水
- 积分操作使用数据库事务保证原子性
- 采纳操作使用行锁（FOR UPDATE）防止并发

### F10: 积分排行榜与流水

**描述**：用户可查看积分排行和个人积分流水。

- 排行榜按积分降序排列，展示全站用户 TOP 100
- 前三名金银铜奖杯图标
- 当前用户高亮显示
- 积分流水按时间倒序，区分收入（绿色+）和支出（红色-）

### F11: 热度榜

**描述**：按浏览量排序的帖子热度排行榜。

- 完全按 `view_count` 降序排列
- 前三名金色/银色/铜色标签

### F12: 帖子搜索

**描述**：用户可以通过关键词搜索帖子。

- 支持标题和内容的模糊匹配（LIKE 查询）
- 搜索关键词最长 50 字符
- 搜索结果按置顶 > 加精 > 时间倒序排列
- 支持分页，无结果时显示友好提示

### F13: AI 智能总结

**描述**：调用大模型 API 生成帖子内容摘要。

- 仅登录用户可使用
- 调用硅基流动 API（`Qwen/Qwen2.5-7B-Instruct`）
- 结果存入数据库，避免重复调用
- 未登录用户看不到 AI 总结结果

### F14: 封面图自动生成

**描述**：无封面图的帖子自动生成 SVG 彩色封面。

- 10 组渐变配色，通过 postId 确定性选择
- 居中显示标题首字符 + 装饰圆形
- 支持高清屏（viewBox 416×256）
- HTTP 缓存一天

### F15: 统一弹窗系统

**描述**：用自定义 CSS 模态框替代所有原生浏览器弹窗。

- 三种模式：蓝色提示（alert）、橙色确认（confirm）、红色错误（error）
- Promise 异步模式，支持 await
- 点击遮罩可关闭确认弹窗

## 4. 非功能性需求

### 4.1 性能需求

- 页面加载时间不超过 2 秒
- 分页查询（LIMIT/OFFSET）支持大数据量场景
- 统计数据 15 秒缓存，减少重复查询
- 封面图 HTTP 缓存 1 天

### 4.2 安全需求

- 密码 BCrypt 加密存储，不存明文
- 所有数据库操作使用 PreparedStatement 防 SQL 注入
- Session 鉴权 Filter 保护敏感路径
- 管理员操作强制 POST 方式
- 敏感配置（数据库密码、API Key）从 config.properties 读取，已加入 .gitignore
- 图片上传白名单（jpg/png/gif/webp）、大小限制（5MB）

### 4.3 可用性需求

- 统一的导航栏 + 侧边栏 + 面包屑导航
- 操作成功/失败浮层通知（URL 参数 + 绿色/红色提示条）
- 所有表单前端 HTML5 校验 + 后端双重校验
- 空数据状态友好提示（"暂无回复"、"还没有帖子"等）

### 4.4 兼容性需求

- 支持 Chrome / Firefox / Edge 现代浏览器
- 使用 Tailwind CSS 响应式类，适配移动端

### 4.5 可维护性需求

- MVC 分层架构（Controller-Service-View）
- 统一工具类（PostMapper/DBUtil/AiUtil）
- 中文注释，关键函数说明用途
- java.util.logging.Logger 记录日志

### 4.6 可靠性需求

- 积分操作使用数据库事务（commit/rollback）
- 关键操作使用行锁（SELECT FOR UPDATE）防止并发异常
- 扣分操作使用原子更新确保积分不为负

## 7. 数据流概述

### 7.1 用户操作数据流

```
用户 → 浏览器操作 → Servlet（doGet/doPost）
    → 参数校验
    → DBUtil 获取数据库连接
    → JDBC 查询/更新（PreparedStatement）
    → 业务处理（积分增减、状态变更等）
    → 设置 request/session 属性
    → JSP 渲染
    → HTTP 响应（HTML 页面或 JSON）
```

### 7.2 积分操作事务流

```
发布需求扣分：
    conn.setAutoCommit(false)
    → INSERT INTO demands (title, content, user_id, score, status)
    → UPDATE users SET score = score - ? WHERE id = ? AND score >= ?
    → INSERT INTO score_logs (user_id, score, reason)
    → conn.commit() 或 catch → conn.rollback()

采纳回复加分：
    conn.setAutoCommit(false)
    → SELECT ... FROM demands WHERE id = ? FOR UPDATE（行锁）
    → 权限/状态校验
    → UPDATE users SET score = score + ? WHERE id = ?
    → INSERT INTO score_logs (user_id, score, reason)
    → UPDATE demands SET status='closed', best_reply_id=? WHERE id=?
    → conn.commit() 或 catch → conn.rollback()
```

## 9. 安全设计

### 9.1 认证安全
- BCrypt 加密存储密码
- 登录后 `changeSessionId()` 防 Session Fixation
- AuthFilter 统一鉴权保护敏感路径

### 9.2 数据安全
- PreparedStatement 参数化查询防 SQL 注入
- 用户输入 HTML 转义防 XSS
- config.properties 存储敏感配置，已 .gitignore

### 9.3 权限控制
- 管理员操作需 `role == 'admin'`
- 帖子编辑/删除限作者或管理员
- 采纳回复限需求发布者
- 需求编辑限发布者且仅 open 状态

## 1. ER图说明

### 1.1 实体关系（文字描述）

```
users(用户) 是系统核心实体，与其他实体的关系如下：

users 1──N posts                — 一个用户可以发布多篇帖子
users 1──N replies              — 一个用户可以发表多条回复
users 1──N demands              — 一个用户可以发布多个需求
users 1──N demand_replies       — 一个用户可以回复多个需求
users 1──N score_logs           — 一个用户有多条积分流水
users 1──N daily_checkins       — 一个用户有多条签到记录
users 1──N user_follows         — 一个用户可以关注多个其他用户
users 1──N user_follows         — 一个用户可以被多个其他用户关注
users 1──N post_likes           — 一个用户可以点赞多篇帖子
users 1──N post_favorites       — 一个用户可以收藏多篇帖子

categories 1──N posts           — 一个板块下有多篇帖子
posts 1──N replies              — 一篇帖子有多条回复
demands 1──N demand_replies     — 一个需求有多条回复
posts 1──N post_likes           — 一篇帖子可以被多个用户点赞
posts 1──N post_favorites       — 一篇帖子可以被多个用户收藏
```

## 2. 数据表详表

系统共包含 **11 张数据表**，涵盖用户、帖子、板块、需求、积分、互动六大类。

---

### 2.1 用户表（users）

**说明**：存储系统所有用户信息，包括认证信息和基本资料。组员B负责。

```sql
CREATE TABLE users (
    id            INT AUTO_INCREMENT PRIMARY KEY COMMENT '用户唯一标识',
    username      VARCHAR(50)  NOT NULL UNIQUE COMMENT '用户名',
    password      VARCHAR(255) NOT NULL COMMENT 'BCrypt加密密码',
    phone         VARCHAR(20)  DEFAULT '' COMMENT '联系方式',
    job_type      VARCHAR(50)  DEFAULT '' COMMENT '工作性质',
    job_location  VARCHAR(100) DEFAULT '' COMMENT '工作地点',
    role          ENUM('user','admin') NOT NULL DEFAULT 'user' COMMENT '角色',
    score         INT NOT NULL DEFAULT 0 COMMENT '积分累计',
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 用户唯一标识 |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR(255) | NOT NULL | BCrypt加密密码（`$2a$10$...`格式） |
| phone | VARCHAR(20) | DEFAULT '' | 联系方式 |
| job_type | VARCHAR(50) | DEFAULT '' | 工作性质 |
| job_location | VARCHAR(100) | DEFAULT '' | 工作地点 |
| role | ENUM('user','admin') | NOT NULL DEFAULT 'user' | 角色 |
| score | INT | NOT NULL DEFAULT 0 | 积分累计 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 注册时间 |

**默认数据**：
```sql
INSERT INTO users (username, password, role, score) VALUES
('admin', 'admin123', 'admin', 0),
('test', 'test123', 'user', 0);
```

> 默认密码为明文，配合后端"首次登录成功后自动升级为 BCrypt"的兼容策略。

### 2.2 板块表（categories）

**说明**：论坛板块分类，用于组织帖子内容。组员C负责。

```sql
CREATE TABLE categories (
    id          INT AUTO_INCREMENT PRIMARY KEY COMMENT '板块唯一标识',
    name        VARCHAR(50)  NOT NULL COMMENT '板块名称',
    description VARCHAR(200) DEFAULT '' COMMENT '板块描述',
    sort_order  INT DEFAULT 0 COMMENT '排序权重',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='板块表';
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 板块唯一标识 |
| name | VARCHAR(50) | NOT NULL | 板块名称 |
| description | VARCHAR(200) | DEFAULT '' | 板块描述 |
| sort_order | INT | DEFAULT 0 | 排序权重 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**默认数据**：
```sql
INSERT INTO categories (name, description, sort_order) VALUES
('技术交流', '编程技术、开发经验分享', 1),
('生活杂谈', '日常生活、兴趣爱好交流', 2),
('求职招聘', '工作机会、求职经验分享', 3),
('需求悬赏', '发布需求、悬赏求助', 4);
```

### 2.3 帖子表（posts）

**说明**：存储论坛帖子内容。组长+组员A负责。

```sql
CREATE TABLE posts (
    id              INT AUTO_INCREMENT PRIMARY KEY COMMENT '帖子唯一标识',
    title           VARCHAR(100) NOT NULL COMMENT '标题',
    content         TEXT NOT NULL COMMENT '内容（Markdown格式）',
    image_url       VARCHAR(500) DEFAULT '' COMMENT '封面图片URL',
    user_id         INT NOT NULL COMMENT '作者ID',
    category_id     INT NOT NULL COMMENT '所属板块ID',
    is_top          TINYINT DEFAULT 0 COMMENT '置顶 0=否 1=板块 2=全局',
    is_elite        TINYINT DEFAULT 0 COMMENT '加精 0=否 1=是',
    ai_summary      TEXT DEFAULT NULL COMMENT 'AI生成总结',
    ai_user_id      INT DEFAULT NULL COMMENT 'AI总结生成者ID',
    keywords        VARCHAR(200) DEFAULT '' COMMENT '关键词（逗号分隔）',
    like_count      INT DEFAULT 0 COMMENT '点赞数',
    favorite_count  INT DEFAULT 0 COMMENT '收藏数',
    view_count      INT DEFAULT 0 COMMENT '浏览次数',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '发布时间',
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (category_id) REFERENCES categories(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='帖子表';
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 帖子唯一标识 |
| title | VARCHAR(100) | NOT NULL | 标题 |
| content | TEXT | NOT NULL | 内容（Markdown格式） |
| image_url | VARCHAR(500) | DEFAULT '' | 封面图片URL |
| user_id | INT | NOT NULL, FK | 作者ID，关联 users.id |
| category_id | INT | NOT NULL, FK | 所属板块ID，关联 categories.id |
| is_top | TINYINT | DEFAULT 0 | 置顶状态：0=否 1=板块置顶 2=全局置顶 |
| is_elite | TINYINT | DEFAULT 0 | 加精状态：0=否 1=是 |
| ai_summary | TEXT | DEFAULT NULL | AI生成的内容总结 |
| ai_user_id | INT | DEFAULT NULL | AI总结生成者ID |
| keywords | VARCHAR(200) | DEFAULT '' | 关键词（逗号分隔） |
| like_count | INT | DEFAULT 0 | 点赞数 |
| favorite_count | INT | DEFAULT 0 | 收藏数 |
| view_count | INT | DEFAULT 0 | 浏览次数 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 发布时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 修改时间 |

### 2.4 回复表（replies）

**说明**：存储帖子的回复内容。组长负责。

```sql
CREATE TABLE replies (

    FOREIGN KEY (post_id) REFERENCES posts(id) ON DELETE CASCADE

```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 回复唯一标识 |
| content | TEXT | NOT NULL | 回复内容 |
| user_id | INT | NOT NULL, FK | 回复者ID，关联 users.id |
| post_id | INT | NOT NULL, FK | 所属帖子ID，关联 posts.id（级联删除） |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 回复时间 |

### 2.5 需求表（demands）

| 字段 | 类型 | 约束 | 说明 |

### 2.7 积分流水表（score_logs）

**说明**：记录所有积分变动的流水明细，用于审计和用户查看。组员D负责。

```sql
CREATE TABLE score_logs (
    id          INT AUTO_INCREMENT PRIMARY KEY COMMENT '流水唯一标识',

    reason      VARCHAR(100) DEFAULT '' COMMENT '变动原因',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '变动时间',
    FOREIGN KEY (user_id) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='积分流水表';
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 流水唯一标识 |
| user_id | INT | NOT NULL, FK | 用户ID，关联 users.id |
| score | INT | NOT NULL | 积分变动（正数=获得，负数=扣除） |
| reason | VARCHAR(100) | DEFAULT '' | 变动原因，如"发帖+10"、"发布悬赏需求，支出20积分" |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 变动时间 |

### 2.8 签到表（daily_checkins）

**说明**：记录用户每日签到信息，支持连续签到奖励递增。组员B负责。

```sql
CREATE TABLE daily_checkins (
    id                INT AUTO_INCREMENT PRIMARY KEY COMMENT '签到记录唯一标识',
    user_id           INT NOT NULL COMMENT '用户ID',
    checkin_date      DATE NOT NULL COMMENT '签到日期',
    consecutive_days  INT NOT NULL DEFAULT 1 COMMENT '连续签到天数',
    score_earned      INT NOT NULL DEFAULT 5 COMMENT '本次获得积分',
    created_at        DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '签到时间',
    FOREIGN KEY (user_id) REFERENCES users(id),
    UNIQUE KEY uk_user_date (user_id, checkin_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='每日签到表';
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 签到记录唯一标识 |
| user_id | INT | NOT NULL, FK | 用户ID，关联 users.id |
| checkin_date | DATE | NOT NULL | 签到日期 |
| consecutive_days | INT | NOT NULL DEFAULT 1 | 连续签到天数 |
| score_earned | INT | NOT NULL DEFAULT 5 | 本次获得积分 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 签到时间 |

**唯一约束**：`uk_user_date (user_id, checkin_date)` — 确保每天每用户只能签到一次

## 3. 表关系说明

### 3.1 一对一关系

- `demands.best_reply_id` —→ `demand_replies.id`（最佳回复与需求的关系，可为空）

### 3.2 一对多关系

| 父表 | 子表 | 外键字段 | 说明 |
|------|------|----------|------|
| users | posts | user_id | 用户 → 帖子 |
| users | replies | user_id | 用户 → 回复 |
| users | demands | user_id | 用户 → 需求 |
| users | demand_replies | user_id | 用户 → 需求回复 |
| users | score_logs | user_id | 用户 → 积分流水 |
| users | daily_checkins | user_id | 用户 → 签到记录 |
| users | user_follows | user_id | 关注者 → 关注记录 |
| users | user_follows | followed_user_id | 被关注者 → 关注记录 |
| categories | posts | category_id | 板块 → 帖子 |
| posts | replies | post_id | 帖子 → 回复 |
| posts | post_likes | post_id | 帖子 → 点赞 |
| posts | post_favorites | post_id | 帖子 → 收藏 |
| demands | demand_replies | demand_id | 需求 → 需求回复 |

### 3.3 级联删除

以下外键设置了 `ON DELETE CASCADE`，删除父表记录时自动删除子表关联记录：

| 父表 | 子表 | 外键字段 |
|------|------|----------|
| posts | replies | post_id |
| posts | post_likes | post_id |
| posts | post_favorites | post_id |
| demands | demand_replies | demand_id |

## 4. 索引设计

### 4.1 主键索引

| 表名 | 索引字段 | 说明 |
|------|----------|------|
| 所有表 | id | 自增主键，集群索引 |

### 4.2 唯一索引

| 表名 | 索引名 | 索引字段 | 说明 |
|------|--------|----------|------|
| users | uk_username | username | 用户名唯一性约束 |
| daily_checkins | uk_user_date | (user_id, checkin_date) | 每天每人限签到一次 |

### 4.3 普通索引

| 表名 | 索引字段 | 说明 |
|------|----------|------|
| posts | user_id | 按作者查询帖子 |
| posts | category_id | 按板块查询帖子 |
| posts | created_at | 按时间排序（默认查询） |
| replies | post_id | 按帖子查询回复 |
| demands | user_id | 按用户查询需求 |
| score_logs | user_id | 按用户查询积分流水 |
| user_follows | user_id | 按关注者查询关注列表 |
| user_follows | followed_user_id | 按被关注者查询粉丝列表 |
| post_likes | user_id | 查用户点赞列表 |
| post_likes | post_id | 查帖子被赞列表 |
| post_favorites | user_id | 查用户收藏列表 |
| post_favorites | post_id | 查帖子被收藏列表 |

## 5. 事务设计

### 5.1 事务使用场景

| 业务场景 | 涉及操作 | 事务隔离 |
|----------|----------|----------|
| 发布帖子 | INSERT posts + UPDATE users.score + INSERT score_logs | READ COMMITTED |
| 回复帖子 | INSERT replies + UPDATE users.score + INSERT score_logs | READ COMMITTED |
| 发布需求 | INSERT demands + UPDATE users.score + INSERT score_logs | READ COMMITTED |
| 采纳回复 | SELECT FOR UPDATE + UPDATE users.score + INSERT score_logs + UPDATE demands | 行锁隔离 |
| 点赞加分 | INSERT/DELETE post_likes + UPDATE posts.like_count + UPDATE users.score + INSERT score_logs | READ COMMITTED |
| 编辑资料改密码 | UPDATE users(profile) + UPDATE users(password) | READ COMMITTED |

### 5.2 事务处理规范

```java
conn.setAutoCommit(false);
try {
    // 1. 核心数据操作（INSERT / UPDATE）
    // 2. 积分相关操作
    // 3. 流水记录
    conn.commit();
} catch (Exception e) {
    conn.rollback();
    throw e;
} finally {
    conn.setAutoCommit(true);
}
```

### 5.3 行锁使用

采纳回复使用 `SELECT ... FOR UPDATE` 防止并发：

```sql
-- 锁定需求行，防止并发采纳
SELECT user_id, score, status FROM demands WHERE id = ? FOR UPDATE;

-- 锁定后校验权限和状态
-- 执行积分更新和状态变更
-- 提交事务释放锁
```

### 5.4 积分原子更新

```sql
-- 扣分（确保积分不为负）
UPDATE users SET score = score - ? WHERE id = ? AND score >= ?;

-- 加分
UPDATE users SET score = score + ? WHERE id = ?;
```

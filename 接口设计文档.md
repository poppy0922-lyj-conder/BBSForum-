## 1. 公共约定

### 1.1 基础路径

```
http://localhost:8088/BBSForum/
```

### 1.2 请求方式

| 方法 | 用途 |
|------|------|
| GET | 页面展示、数据查询 |
| POST | 数据提交、状态变更 |

### 1.3 数据格式

- **页面请求**：返回 JSP 页面（HTML + Tailwind CSS）
- **JSON 请求**：交互类接口返回 `Content-Type: application/json`
- **图片请求**：封面图生成返回 `image/svg+xml`

### 1.4 通用参数

| 参数 | 说明 |
|------|------|
| `success` | URL参数，操作成功标识（1=成功） |
| `error` | URL参数，操作失败标识（1=失败，或错误描述） |
| `page` | URL参数，分页页码（从1开始，默认1） |
| `refresh` | URL参数，强制刷新缓存（1=刷新） |

## 2. 帖子模块 API

### 2.1 首页帖子列表

```
GET /
GET /index
GET /home
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `page` | int | 否 | 页码，默认1 |
| `refresh` | int | 否 | 刷新板块缓存 |

**返回**：首页 JSP 页面，包含帖子列表、板块导航、右侧数据面板

**排序规则**：`is_top DESC（全局置顶 > 板块置顶 > 普通）` → `is_elite DESC` → `created_at DESC`

**注意**：所有帖子查询均过滤 `is_deleted = 0`，已软删除的帖子不可见。

### 2.2 帖子详情

```
GET /post/detail?id={id}
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `id` | int | 是 | 帖子ID |

**返回**：帖子详情 JSP 页面
- 帖子内容（Markdown 渲染为 HTML）
- AI 总结区域（仅登录用户可见）
- 回复列表（按时间正序，过滤 is_deleted=0）
- 相关推荐（右侧栏，最多5篇）
- 互动按钮（关注/点赞/收藏，显示当前登录用户交互状态）

### 2.3 发布帖子

```
GET /post/create      # 显示发帖表单
POST /post/create     # 提交新帖
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `title` | string | 是 | 帖子标题，最大100字符 |
| `content` | string | 是 | 帖子内容（Markdown） |
| `categoryId` | int | 是 | 所属板块ID |
| `keywords` | string | 否 | 关键词，逗号分隔 |
| `coverImage` | file | 否 | 封面图文件（jpg/png/gif/webp ≤5MB） |
| `imageUrl` | string | 否 | 封面图URL |

**返回**：成功 → 重定向到新帖详情页，失败 → 返回表单页显示错误

**积分规则**：发帖成功后，发帖者积分 **+10**

### 2.4 编辑帖子

```
GET /post/edit?id={id}      # 显示编辑表单
POST /post/edit             # 提交编辑
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `id` | int | 是 | 帖子ID |
| `title` | string | 是 | 标题 |
| `content` | string | 是 | 内容 |
| `categoryId` | int | 是 | 板块ID |
| `keywords` | string | 否 | 关键词 |
| `coverImage` | file | 否 | 新封面图 |
| `imageUrl` | string | 否 | 封面图URL |

**权限**：帖子作者 或 管理员（role=admin）

### 2.5 删除帖子（软删除）

```
POST /post/delete?id={id}
```

**参数**：`id`（帖子ID）

**权限**：帖子作者 或 管理员

**说明**：采用软删除机制，执行 `UPDATE posts SET is_deleted = 1 WHERE id = ?`，非实际物理删除。
被删除的帖子对所有用户不可见（包括管理员），所有查询均过滤 `is_deleted = 0`。

### 2.6 回复帖子

```
POST /post/reply
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `postId` | int | 是 | 帖子ID |
| `content` | string | 是 | 回复内容 |
| `parentId` | int | 否 | 父回复ID，用于嵌套回复 |

**嵌套回复说明**：
- 传 `parentId` 表示回复指定回复，不传表示直接回复帖子
- 被回复用户收到 `reply_reply` 类型通知，附带 target_url 指向帖子
- 子回复在父回复下方缩进显示

**返回**：重定向回帖子详情页

**积分规则**：回复成功后，回复者积分 **+2**

### 2.7 AI 总结

```
POST /post/aiSummary
```

**参数**：`id`（帖子ID）

**返回**：JSON

```json
// 成功
{"summary": "这篇帖子介绍了Java 21虚拟线程的使用方法..."}

// 失败
{"error": "生成失败，请稍后重试"}
```

**说明**：
- 需登录
- 调用硅基流动 API（`Qwen/Qwen2.5-7B-Instruct`）
- 结果存入 `posts.ai_summary` + 记录生成者 `ai_user_id`，下次直接读取

### 2.8 帖子搜索

```
GET /post/search?keyword={keyword}&page={page}
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `keyword` | string | 是 | 搜索关键词，最大50字符 |
| `page` | int | 否 | 页码 |

**返回**：搜索结果 JSP 页面
- 标题头显示搜索关键词和结果数量
- 排序规则：置顶 > 加精 > 时间倒序
- 匹配范围：标题、内容、关键词（LIKE 模糊匹配）
- 过滤 `is_deleted = 0`

### 2.9 封面图生成

```
GET /cover/{postId}?title={title}
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `postId` | int | 路径参数 | 帖子ID，用于确定配色方案 |
| `title` | string | 否 | 帖子标题，取首字符显示 |

**返回**：`Content-Type: image/svg+xml`

**说明**：
- 无封面图的帖子自动调用
- 10组渐变配色，通过 `postId % 10` 确定
- 装饰圆形位置通过 `Random(postId ^ 0xABCDEF)` 确定性生成
- `Cache-Control: max-age=86400`

### 2.10 内联图片上传

```
POST /post/uploadImage
```

**参数**：`coverImage`（file，需登录）

**返回**：JSON

```json
{"markdown": "![图片](/BBSForum/uploads/xxx.jpg)", "url": "/BBSForum/uploads/xxx.jpg"}
```

### 2.11 帖子草稿箱

```
POST /post/create
```

**POST 参数**（新增 action 字段）：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `title` | string | 是 | 帖子标题 |
| `content` | string | 是 | 帖子内容 |
| `categoryId` | int | 是 | 所属板块ID |
| `keywords` | string | 否 | 关键词 |
| `action` | string | 否 | `publish`（发布，默认）或 `draft`（保存草稿） |

```
GET /post/drafts
```

**说明**：需登录，查看当前用户所有草稿（is_draft=1）

**返回**：草稿箱 JSP 页面，列出所有草稿，每条支持继续编辑和删除

**草稿说明**：
- `action=draft` 时保存为草稿，is_draft=1，不发放+10积分
- 草稿在首页、板块页、搜索中不可见（过滤 is_draft=0）
- 编辑草稿发布时 is_draft 置 0，发放+10积分

## 3. 用户模块 API

### 3.1 用户注册

```
GET /user/register      # 显示注册表单
POST /user/register     # 提交注册
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `username` | string | 是 | 用户名，唯一 |
| `password` | string | 是 | 密码 |
| `password2` | string | 是 | 确认密码 |
| `phone` | string | 否 | 联系方式 |
| `jobType` | string | 否 | 工作性质 |
| `jobLocation` | string | 否 | 工作地点 |

**返回**：
- 成功：重定向到 `/user/login?registered=1`
- 失败：返回注册页并显示错误（用户名重复、两次密码不一致等）

**说明**：密码使用 BCrypt 加密存储；注册时写入积分为 0 的流水记录（reason: "新用户注册"）

### 3.2 用户登录

```
GET /user/login      # 显示登录表单
POST /user/login     # 提交登录
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `username` | string | 是 | 用户名 |
| `password` | string | 是 | 密码 |

**返回**：
- 成功：建立 Session，重定向到首页
- 失败：返回登录页显示错误

**Session 结构**：`sessionScope.user` → `Map{id, username, role, phone, jobType, jobLocation, avatar, score, createdAt}`

**积分规则**：登录成功后，当天首次登录积分 **+2**

**兼容说明**：支持 BCrypt hash 和旧明文密码的双重校验，旧明文首次登录自动升级为 BCrypt

### 3.3 退出登录

```
GET /logout
```

**说明**：清除当前 Session，重定向到首页

### 3.4 个人中心

```
GET /user/profile
```

**说明**：需登录（AuthFilter 保护）

**返回**：个人中心 JSP 页面
- 用户基本信息（用户名、联系方式、工作性质、工作地点、头像、注册时间）
- 当前积分
- 最近5条积分流水记录（边栏展示）

### 3.5 编辑资料（含头像上传）

```
GET /user/profile/edit      # 显示编辑表单
POST /user/profile/edit     # 提交编辑
```

**说明**：支持 `multipart/form-data` 格式（`@MultipartConfig`），支持头像上传。

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `phone` | string | 否 | 联系方式 |
| `jobType` | string | 否 | 工作性质 |
| `jobLocation` | string | 否 | 工作地点 |
| `password` | string | 否 | 新密码（不修改留空） |
| `password2` | string | 否 | 确认新密码 |
| `avatar` | file | 否 | 头像图片文件（jpg/png/gif/webp ≤5MB） |

**头像上传说明**：
- 支持格式：JPEG、PNG、GIF、WebP
- 大小限制：≤5MB
- 存储路径：`{webapp}/uploads/avatars/user_{userId}_{uuid}.{ext}`
- 上传新头像会自动删除旧头像文件
- 数据库存储相对URL，如 `/BBSForum/uploads/avatars/user_1_xxx.jpg`

**返回**：成功 → 重定向到 `/user/profile?updated=1`

### 3.6 每日签到

```
GET /user/checkin
```

**说明**：需登录

**返回**：JSON

```json
{"ok": true, "score": 6, "consecutive": 3}
{"ok": false, "msg": "今日已签到"}
```

**积分规则**：
- 每天限签到一次
- 连续签到递增：第1天+5，第2天+6，第3天+7...封顶+15
- 断签重置为第1天

### 3.7 关注列表

```
GET /user/profile/follows
```

**说明**：需登录，查看当前用户关注的所有用户

**返回**：关注列表 JSP 页面，显示已关注用户的用户名、联系方式、工作性质、工作地点

### 3.8 积分记录

```
GET /user/score-log?page={page}
```

**说明**：需登录，查看当前用户积分流水记录（分页）

**参数**：`page`（int，可选，默认1）

**返回**：积分记录 JSP 页面，每页15条，按时间倒序排列

### 3.9 个人中心子页面

以下子页面从个人中心边栏访问，均需登录：

| 接口 | 功能 | 说明 |
|------|------|------|
| `/user/profile/posts` | 我的发布 | 显示当前用户发布的所有帖子列表 |
| `/user/profile/demands` | 我的悬赏 | 显示当前用户发布的所有需求列表 |
| `/user/profile/likes` | 我的点赞 | 显示当前用户点赞过的所有帖子列表 |
| `/user/profile/favorites` | 我的收藏 | 显示当前用户收藏过的所有帖子列表 |

## 4. 交互模块 API

### 4.1 关注/取消关注

```
POST /interact/follow
```

**参数**：`userId`（int，被关注者ID）

**返回**：JSON

```json
{"ok": true, "action": "follow"}
{"ok": true, "action": "unfollow"}
{"ok": false, "msg": "不能关注自己"}
```

**说明**：切换操作，已关注则取消，未关注则关注

### 4.2 点赞/取消点赞

```
POST /interact/like
```

**参数**：`postId`（int，帖子ID）

**返回**：JSON

```json
{"ok": true, "action": "like", "count": 10}
{"ok": true, "action": "unlike", "count": 9}
```

**积分规则**：
- 点赞时，帖子作者积分 **+3**（reason: "帖子被点赞"）
- 取消点赞时，帖子作者积分 **-3**（reason: "帖子被取消点赞"）

### 4.3 收藏/取消收藏

```
POST /interact/favorite
```

**参数**：`postId`（int，帖子ID）

**返回**：JSON

```json
{"ok": true, "action": "favorite", "count": 5}
{"ok": true, "action": "unfavorite", "count": 4}
```

**积分规则**：
- 收藏时，帖子作者积分 **+5**（reason: "帖子被收藏"）
- 取消收藏时，帖子作者积分 **-5**（reason: "帖子被取消收藏"）

## 5. 需求悬赏模块 API

### 5.1 需求列表

```
GET /demand?page={page}
```

**参数**：`page`（int，可选，默认1）

**返回**：需求列表 JSP 页面
- 每页显示 20 条
- 按发布时间倒序
- 过滤 `is_deleted = 0`
- 已结束的需求显示灰色（`opacity-60`）
- 当前用户自己的需求显示"我的"标识

### 5.2 需求详情

```
GET /demand/detail?id={id}
```

**参数**：`id`（int，需求ID）

**返回**：需求详情 JSP 页面
- 需求信息（标题、内容、悬赏积分、状态）
- 回复列表（过滤 `is_deleted = 0`，含已采纳标识）
- 采纳按钮（仅发布者可见，进行中状态）
- 回复表单（登录用户可见，进行中状态）

### 5.3 发布需求

```
GET /demand/create      # 显示发布表单
POST /demand/create     # 提交发布
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `title` | string | 是 | 需求标题 |
| `content` | string | 是 | 需求描述 |
| `score` | int | 是 | 悬赏积分（>0，≤10000） |

**积分规则**：
- 发布时检查用户当前积分 ≥ 悬赏积分
- 从发布者账户扣除悬赏积分
- 写入 `score_logs`（reason: "发布悬赏需求，支出X积分"）

**事务保证**：`conn.setAutoCommit(false)` → INSERT demand → UPDATE users SET score = score - ? → INSERT score_logs → commit

### 5.4 需求回复

```
POST /demand/reply
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `demandId` | int | 是 | 需求ID |
| `content` | string | 是 | 回复内容 |

**返回**：重定向回需求详情页

**说明**：回复存入 `demand_replies` 表（独立于 `replies` 表）

### 5.5 采纳回复

```
POST /demand/accept
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `demandId` | int | 是 | 需求ID |
| `replyId` | int | 是 | 回复ID |

**权限校验**：
- 仅需求发布者可操作
- 仅状态为 `open`（进行中）的需求可操作（过滤 `is_deleted = 0`）
- 不能采纳自己的回复
- 不能重复采纳（`best_reply_id` 已存在时拒绝）

**返回**：成功 → `location.reload()`（页面刷新），失败 → `alert('操作失败')`

**积分规则**：
- 给被采纳者增加悬赏积分
- 写入 `score_logs`（reason: "回复被采纳，获得悬赏X积分"）
- 需求状态变为 `closed`

**事务保证**：`SELECT ... FOR UPDATE` → UPDATE users SET score = score + ? → INSERT score_logs → UPDATE demands SET status='closed', best_reply_id=? → commit

### 5.6 编辑需求

```
GET /demand/update?id={id}      # 显示编辑表单
POST /demand/update             # 提交编辑
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `id` | int | 是 | 需求ID |
| `title` | string | 是 | 标题 |
| `content` | string | 是 | 内容 |

**权限**：仅需求发布者可操作，仅 `open` 状态可编辑

## 6. 积分模块 API

### 6.1 积分排行榜

```
GET /score/rank
```

**返回**：排行榜 JSP 页面
- 按 `users.score` 降序排列
- 前三名金银铜奖杯图标
- 当前用户高亮显示
- 每页展示所有用户（不分页，TOP 100）

### 6.2 积分记录

```
GET /score/record
```

**说明**：需登录

**返回**：积分流水 JSP 页面
- 按 `created_at` 降序排列
- 显示：积分变动值（正数为绿色+，负数为红色-）、变动原因、变动时间

### 6.3 积分规则汇总

| 操作 | 积分变动 | 触发时机 |
|------|:--------:|----------|
| 发布帖子 | +10 | `PostServlet.createPost` 成功后 |
| 回复帖子 | +2 | `PostServlet.reply` 成功后 |
| 每日签到 | +5 ~ +15 | `UserServlet.checkin` |
| 帖子被点赞 | +3 | `InteractionServlet.like` 时给帖子作者 |
| 帖子被取消点赞 | -3 | `InteractionServlet.like` 取消时给帖子作者 |
| 帖子被收藏 | +5 | `InteractionServlet.favorite` 时给帖子作者 |
| 帖子被取消收藏 | -5 | `InteractionServlet.favorite` 取消时给帖子作者 |
| 每日首次登录 | +2 | `UserServlet.login` 成功时 |
| 回复被采纳 | +悬赏分 | `DemandServlet.accept` 时 |
| 发布需求 | -悬赏分 | `DemandServlet.createDemand` 时 |
| 用户注册 | 0（写入流水） | `UserServlet.register` 时 |

## 7. 管理后台 API

### 7.1 管理员首页

```
GET /admin
```

**说明**：需 admin 角色（AuthFilter 保护）

**返回**：管理员后台首页 JSP 页面

### 7.2 帖子管理

## 8. 举报系统 API

### 8.1 用户提交举报

```
POST /report/submit
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `targetType` | string | 是 | 举报类型：post/reply/demand/demand_reply |
| `targetId` | int | 是 | 被举报对象ID |
| `reason` | string | 是 | 举报原因：spam（垃圾广告）/abuse（人身攻击）/illegal（政治敏感）/porn（色情低俗）/other（其他） |

**权限**：需登录

**返回**：JSON

```json
{"success": true, "message": "举报已提交，感谢您的反馈"}
{"success": false, "message": "您已举报过此内容，请等待处理"}
{"success": false, "message": "不能举报自己"}
{"success": false, "message": "内容不存在或已删除"}
```

**说明**：
- 不能举报自己
- 同一用户对同一内容的待处理（pending）举报仅允许一次

### 8.2 管理员举报列表

```
GET /admin/report/list?status={status}&page={page}
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `status` | string | 否 | 筛选状态：pending/all/approved/rejected，默认 pending |
| `page` | int | 否 | 页码，默认1 |

**权限**：需 admin 角色

**返回**：举报管理 JSP 页面 `/admin/report_manage.jsp`
- 举报列表（每页15条）：ID、举报人、举报类型、对象ID、原因、状态、时间
- 预览被举报内容、处理按钮

### 8.3 管理员预览被举报内容

```
GET /admin/report/preview?reportId={id}
```

**参数**：`reportId`（int，举报记录ID）

**权限**：需 admin 角色

**返回**：HTML 片段，显示被举报内容的类型、标题、内容（截取前200字）、作者、当前状态

### 8.4 管理员处理举报

```
POST /admin/report/handle
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `reportId` | int | 是 | 举报记录ID |
| `action` | string | 是 | 处理动作：approve（通过）/ reject（驳回） |
| `handleNote` | string | 否 | 处理备注，最长200字 |

**权限**：需 admin 角色

**处理逻辑**（事务保证）：
1. `SELECT ... FOR UPDATE` 锁定举报记录
2. 校验当前状态为 `pending`
3. 若通过（approve）：对目标内容执行 `SET is_deleted = 1`
4. 更新举报状态和 handleNote
5. 插入通知：
   - 通知举报人：`report_result` 类型
   - 通知被举报人（仅通过时）：`content_deleted` 类型

## 9. 通知系统 API

### 9.1 未读通知数

```
GET /notification/unreadCount
```

**返回**：JSON

```json
{"count": 3}
```

**说明**：未登录时返回 `{"count": 0}`，用于导航栏小红点显示
- 支持通知类型：report_result, content_deleted, new_reply, new_like, new_favorite, reply_reply

### 9.2 通知列表

```
GET /notification/list?page={page}
```

**参数**：`page`（int，可选，默认1）

**权限**：需登录

**返回**：通知列表 JSP 页面 `/notification/list.jsp`
- 分页显示通知列表（每页20条）
- 显示未读计数
- 通知项：类型标识、内容摘要、时间、target_url（点击直接跳转帖子）
- 通知类型：report_result（举报结果）、content_deleted（内容删除）、new_reply（新回复）、new_like（新点赞）、new_favorite（新收藏）、reply_reply（回复回复）

### 9.3 标记已读

```
POST /notification/markRead?id={id}
```

**参数**：`id`（int，可选，通知ID，不传则标记全部已读）

**说明**：传 id 标记单条，不传 id 标记当前用户全部通知为已读

**返回**：重定向到通知列表页

### 9.4 通知详情

```
GET /notification/detail?id={id}
```

**参数**：`id`（int，通知ID）

**权限**：需登录，仅能查看自己的通知

**返回**：通知详情 JSP 页面 `/notification/detail.jsp`
- 通知内容
- 自动标记为已读
- 关联的举报记录详情（如通知由举报处理触发）
- 如果通知包含 target_url，显示"查看相关帖子"按钮
- 通知列表页优先使用 target_url 跳转，无 target_url 时进入通知详情页

### B. 公共状态码

| HTTP状态码 | 含义 |
|:----------:|------|
| 200 | 成功 |
| 302 | 重定向（页面跳转） |
| 403 | 禁止访问（非管理员访问后台） |
| 400 | 参数错误（如非法postId） |
| 404 | 资源不存在 |

### C. 分页参数

所有分页接口通用规则：
- 默认每页 20 条（举报管理 15 条，积分记录 15 条）
- 页码从 1 开始
- 无效页码自动回退到第 1 页
- 通过 `totalPages` 和 `currentPage` 渲染分页导航

### D. AuthFilter 保护路径

```java
@WebFilter(urlPatterns = {"/user/profile", "/user/profile/*", "/user/score-log", "/admin", "/admin/*"})
```

说明：
- `/user/profile` 及子路径需要登录
- `/admin` 及子路径需要登录且角色为 admin
- 未登录自动跳转 `/user/login`
- 非管理员访问 `/admin` 返回 403

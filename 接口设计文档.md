## 1. 公共约定

### 1.1 基础路径

```
http://localhost:8088/BBSForum/
```

### 1.2 请求方式

| 方法 | 用途 |
|------|------|
| GET | 页面展示、数据查询 |
| POST | 数据提交、状态变更 |

### 1.3 数据格式

- **页面请求**：返回 JSP 页面（HTML + Tailwind CSS）
- **JSON 请求**：交互类接口返回 `Content-Type: application/json`
- **图片请求**：封面图生成返回 `image/svg+xml`

### 1.4 通用参数

| 参数 | 说明 |
|------|------|
| `success` | URL参数，操作成功标识（1=成功） |
| `error` | URL参数，操作失败标识（1=失败，或错误描述） |
| `page` | URL参数，分页页码（从1开始，默认1） |
| `refresh` | URL参数，强制刷新缓存（1=刷新） |

## 2. 帖子模块 API

### 2.1 首页帖子列表

```
GET /
GET /index
GET /home
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `page` | int | 否 | 页码，默认1 |
| `refresh` | int | 否 | 刷新板块缓存 |

**返回**：首页 JSP 页面，包含帖子列表、板块导航、右侧数据面板

**排序规则**：`is_top DESC（全局置顶 > 板块置顶 > 普通）` → `is_elite DESC` → `created_at DESC`

### 2.2 帖子详情

```
GET /post/detail?id={id}
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `id` | int | 是 | 帖子ID |

**返回**：帖子详情 JSP 页面
- 帖子内容（Markdown 渲染为 HTML）
- AI 总结区域（仅登录用户可见）
- 回复列表（按时间正序）
- 相关推荐（右侧栏，最多5篇）
- 互动按钮（关注/点赞/收藏）

### 2.3 发布帖子

```
GET /post/create      # 显示发帖表单
POST /post/create     # 提交新帖
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `title` | string | 是 | 帖子标题，最大100字符 |
| `content` | string | 是 | 帖子内容（Markdown） |
| `categoryId` | int | 是 | 所属板块ID |
| `keywords` | string | 否 | 关键词，逗号分隔 |
| `coverImage` | file | 否 | 封面图文件（jpg/png/gif/webp ≤5MB） |
| `imageUrl` | string | 否 | 封面图URL |

**返回**：成功 → 重定向到新帖详情页，失败 → 返回表单页显示错误

**积分规则**：发帖成功后，发帖者积分 **+10**

### 2.4 编辑帖子

```
GET /post/edit?id={id}      # 显示编辑表单
POST /post/edit             # 提交编辑
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `id` | int | 是 | 帖子ID |
| `title` | string | 是 | 标题 |
| `content` | string | 是 | 内容 |
| `categoryId` | int | 是 | 板块ID |
| `keywords` | string | 否 | 关键词 |
| `coverImage` | file | 否 | 新封面图 |
| `imageUrl` | string | 否 | 封面图URL |

**权限**：帖子作者 或 管理员（role=admin）

### 2.5 删除帖子

```
POST /post/delete?id={id}
```

**参数**：`id`（帖子ID）

**权限**：帖子作者 或 管理员

**说明**：级联删除该帖子的所有回复（ON DELETE CASCADE）

### 2.6 回复帖子

```
POST /post/reply
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `postId` | int | 是 | 帖子ID |
| `content` | string | 是 | 回复内容 |

**返回**：重定向回帖子详情页

**积分规则**：回复成功后，回复者积分 **+2**

### 2.7 AI 总结

```
POST /post/aiSummary
```

**参数**：`id`（帖子ID）

**返回**：JSON

```json
// 成功
{"summary": "这篇帖子介绍了Java 21虚拟线程的使用方法..."}

// 失败
{"error": "生成失败，请稍后重试"}
```

**说明**：
- 需登录
- 调用硅基流动 API（`Qwen/Qwen2.5-7B-Instruct`）
- 结果存入 `posts.ai_summary`，下次直接读取

### 2.8 帖子搜索

```
GET /post/search?keyword={keyword}&page={page}
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `keyword` | string | 是 | 搜索关键词，最大50字符 |
| `page` | int | 否 | 页码 |

**返回**：搜索结果 JSP 页面
- 标题头显示搜索关键词和结果数量
- 排序规则：置顶 > 加精 > 时间倒序

### 2.9 封面图生成

```
GET /cover/{postId}?title={title}
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `postId` | int | 路径参数 | 帖子ID，用于确定配色方案 |
| `title` | string | 否 | 帖子标题，取首字符显示 |

**返回**：`Content-Type: image/svg+xml`

**说明**：
- 无封面图的帖子自动调用
- 10组渐变配色，通过 `postId % 10` 确定
- 装饰圆形位置通过 `Random(postId ^ 0xABCDEF)` 确定性生成
- `Cache-Control: max-age=86400`

### 2.10 内联图片上传

```
POST /post/uploadImage
```

**参数**：`coverImage`（file，需登录）

**返回**：JSON

```json
{"markdown": "![图片](/BBSForum/uploads/xxx.jpg)", "url": "/BBSForum/uploads/xxx.jpg"}
```

## 3. 用户模块 API

### 3.1 用户注册

```
GET /user/register      # 显示注册表单
POST /user/register     # 提交注册
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `username` | string | 是 | 用户名，唯一 |
| `password` | string | 是 | 密码 |
| `password2` | string | 是 | 确认密码 |
| `phone` | string | 否 | 联系方式 |
| `jobType` | string | 否 | 工作性质 |
| `jobLocation` | string | 否 | 工作地点 |

**返回**：
- 成功：重定向到 `/user/login?registered=1`
- 失败：返回注册页并显示错误（用户名重复、两次密码不一致等）

**说明**：密码使用 BCrypt 加密存储

### 3.2 用户登录

```
GET /user/login      # 显示登录表单
POST /user/login     # 提交登录
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `username` | string | 是 | 用户名 |
| `password` | string | 是 | 密码 |

**返回**：
- 成功：建立 Session，重定向到首页
- 失败：返回登录页显示错误

**Session 结构**：`sessionScope.user` → `Map{id, username, role, phone, jobType, jobLocation, createdAt}`

**积分规则**：登录成功后，当天首次登录积分 **+2**

**兼容说明**：支持 BCrypt hash 和旧明文密码的双重校验，旧明文首次登录自动升级为 BCrypt

### 3.3 退出登录

```
GET /logout
```

**说明**：清除当前 Session，重定向到首页

### 3.4 个人中心

```
GET /user/profile
```

**说明**：需登录（AuthFilter 保护）

**返回**：个人中心 JSP 页面
- 用户基本信息（用户名、联系方式、工作性质、工作地点、注册时间）
- 当前积分

### 3.5 编辑资料

```
GET /user/profile/edit      # 显示编辑表单
POST /user/profile/edit     # 提交编辑
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `phone` | string | 否 | 联系方式 |
| `jobType` | string | 否 | 工作性质 |
| `jobLocation` | string | 否 | 工作地点 |
| `password` | string | 否 | 新密码（不修改留空） |
| `password2` | string | 否 | 确认新密码 |

**返回**：成功 → 重定向到 `/user/profile?updated=1`

### 3.6 每日签到

```
GET /user/checkin
```

**说明**：需登录

**返回**：JSON

```json
{"ok": true, "score": 6, "consecutive": 3}
{"ok": false, "msg": "今日已签到"}
```

**积分规则**：
- 每天限签到一次
- 连续签到递增：第1天+5，第2天+6，第3天+7...封顶+15
- 断签重置为第1天

### 3.7 关注列表

```
GET /user/profile/follows
```

**说明**：需登录，查看当前用户关注的所有用户

**返回**：关注列表 JSP 页面

## 4. 交互模块 API

### 4.1 关注/取消关注

```
POST /interact/follow
```

**参数**：`userId`（int，被关注者ID）

**返回**：JSON

```json
{"ok": true, "action": "follow"}
{"ok": true, "action": "unfollow"}
{"ok": false, "msg": "不能关注自己"}
```

**说明**：切换操作，已关注则取消，未关注则关注

### 4.2 点赞/取消点赞

```
POST /interact/like
```

**参数**：`postId`（int，帖子ID）

**返回**：JSON

```json
{"ok": true, "action": "like", "count": 10}
{"ok": true, "action": "unlike", "count": 9}
```

**积分规则**：点赞时，帖子作者积分 **+3**（取消点赞不扣回）

### 4.3 收藏/取消收藏

```
POST /interact/favorite
```

**参数**：`postId`（int，帖子ID）

**返回**：JSON

```json
{"ok": true, "action": "favorite", "count": 5}
{"ok": true, "action": "unfavorite", "count": 4}
```

## 5. 需求悬赏模块 API

### 5.1 需求列表

```
GET /demand?page={page}
```

**参数**：`page`（int，可选，默认1）

**返回**：需求列表 JSP 页面
- 每页显示 20 条
- 按发布时间倒序
- 已结束的需求显示灰色（`opacity-60`）
- 当前用户自己的需求显示"我的"标识

### 5.2 需求详情

```
GET /demand/detail?id={id}
```

**参数**：`id`（int，需求ID）

**返回**：需求详情 JSP 页面
- 需求信息（标题、内容、悬赏积分、状态）
- 回复列表（含已采纳标识）
- 采纳按钮（仅发布者可见，进行中状态）
- 回复表单（登录用户可见，进行中状态）

### 5.3 发布需求

```
GET /demand/create      # 显示发布表单
POST /demand/create     # 提交发布
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `title` | string | 是 | 需求标题 |
| `content` | string | 是 | 需求描述 |
| `score` | int | 是 | 悬赏积分（>0，≤10000） |

**积分规则**：
- 发布时检查用户当前积分 ≥ 悬赏积分
- 从发布者账户扣除悬赏积分
- 写入 `score_logs`（reason: "发布悬赏需求，支出X积分"）

**事务保证**：`conn.setAutoCommit(false)` → INSERT demand → UPDATE users SET score = score - ? → INSERT score_logs → commit

### 5.4 需求回复

```
POST /demand/reply
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `demandId` | int | 是 | 需求ID |
| `content` | string | 是 | 回复内容 |

**返回**：重定向回需求详情页

**说明**：回复存入 `demand_replies` 表（独立于 `replies` 表）

### 5.5 采纳回复

```
POST /demand/accept
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `demandId` | int | 是 | 需求ID |
| `replyId` | int | 是 | 回复ID |

**权限校验**：
- 仅需求发布者可操作
- 仅状态为 `open`（进行中）的需求可操作
- 不能采纳自己的回复
- 不能重复采纳（`best_reply_id` 已存在时拒绝）

**返回**：成功 → `location.reload()`（页面刷新），失败 → `alert('操作失败')`

**积分规则**：
- 给被采纳者增加悬赏积分
- 写入 `score_logs`（reason: "回复被采纳，获得悬赏X积分"）
- 需求状态变为 `closed`

**事务保证**：`SELECT ... FOR UPDATE` → UPDATE users SET score = score + ? → INSERT score_logs → UPDATE demands SET status='closed', best_reply_id=? → commit

### 5.6 编辑需求

```
GET /demand/update?id={id}      # 显示编辑表单
POST /demand/update             # 提交编辑
```

**POST 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `id` | int | 是 | 需求ID |
| `title` | string | 是 | 标题 |
| `content` | string | 是 | 内容 |

**权限**：仅需求发布者可操作，仅 `open` 状态可编辑

## 6. 积分模块 API

### 6.1 积分排行榜

```
GET /score/rank
```

**返回**：排行榜 JSP 页面
- 按 `users.score` 降序排列
- 前三名金银铜奖杯图标
- 当前用户高亮显示
- 每页展示所有用户（不分页，TOP 100）

### 6.2 积分记录

```
GET /score/record
```

**说明**：需登录

**返回**：积分流水 JSP 页面
- 按 `created_at` 降序排列
- 显示：积分变动值（正数为绿色+，负数为红色-）、变动原因、变动时间

### 6.3 积分规则汇总

| 操作 | 积分变动 | 触发时机 |
|------|:--------:|----------|
| 发布帖子 | +10 | `PostServlet.createPost` 成功后 |
| 回复帖子 | +2 | `PostServlet.reply` 成功后 |
| 每日签到 | +5 ~ +15 | `UserServlet.checkin` |
| 帖子被点赞 | +3 | `InteractionServlet.like` 时 |
| 每日首次登录 | +2 | `UserServlet.login` 成功时 |
| 回复被采纳 | +悬赏分 | `DemandServlet.accept` 时 |
| 发布需求 | -悬赏分 | `DemandServlet.createDemand` 时 |

## 7. 管理后台 API

```
GET /admin
```

**说明**：需 admin 角色（AuthFilter 保护）

**返回**：管理员后台首页

```
GET /admin/categories
```

| 接口 | 功能 | 参数 |
|------|------|------|

| `/admin/categories/add` | 添加板块 | name, description |
| `/admin/categories/edit` (GET) | 编辑表单 | id |
| `/admin/categories/edit` (POST) | 提交编辑 | id, name, description |

## 附录

### A. Session 结构

```java
// sessionScope.user 的 Map 结构
{
    "id": 1,              // int, 用户ID
    "username": "admin",  // String, 用户名
    "role": "admin",      // String, 角色（user 或 admin）
    "phone": "138xxx",    // String, 联系方式
    "jobType": "学生",    // String, 工作性质
    "jobLocation": "北京", // String, 工作地点
    "createdAt": "2026-06-06" // String, 注册时间
}
```

### B. 公共状态码

| HTTP状态码 | 含义 |
|:----------:|------|
| 200 | 成功 |
| 302 | 重定向（页面跳转） |
| 403 | 禁止访问（非管理员访问后台） |
| 400 | 参数错误（如非法postId） |
| 404 | 资源不存在 |

### C. 分页参数

所有分页接口通用规则：
- 默认每页 20 条
- 页码从 1 开始
- 无效页码自动回退到第 1 页
- 通过 `totalPages` 和 `currentPage` 渲染分页导航
